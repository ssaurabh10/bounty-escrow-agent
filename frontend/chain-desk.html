Chain desk

On-chain controls for the deployed BountyEscrow app. Serve the repository root (not file://) so contract.json and deploy_info.json load. Example: npx serve . then open /frontend/chain-desk.html.

Contract state

Connect to LocalNet and set App ID, then refresh.

Post bounty (on-chain)

Uses the selected signer as creator. Reward in whole ALGO (converted to microAlgos). Deadline = now + days (Unix).

Reopen after rejection (creator)

Calls reopen_after_rejection when status is REJECTED, the dispute window has ended (5 min on LocalNet demo; same as AUTO_RELEASE_DELAY), and the bounty deadline has not passed. Escrow stays in the app; another account can accept_bounty.
